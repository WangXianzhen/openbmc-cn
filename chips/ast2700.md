# ASPEED AST2700 介绍

> 最新的服务器级 BMC 芯片

## 📋 芯片概述

ASPEED AST2700 是 ASPEED 推出的下一代服务器 BMC 芯片，具备：

- **双核 ARM Cortex-A7 或 ARM Cortex-A35**
- **支持双 Flash 启动**
- **更高性能的视频处理能力**
- **增强的安全特性**

## 🆚 芯片对比

| 特性 | AST2600 | AST2700 |
|------|---------|---------|
| CPU | 单核/双核 Cortex-A7 | 双核 Cortex-A7/A35 |
| 主频 | 1.2 GHz | 1.4 GHz |
| DDR | DDR3/DDR4 | DDR4/DDR5 |
| 视频 | 1920x1200 | 4K |
| USB | USB 2.0 | USB 3.2 |
| PCIe | Gen2 | Gen3 |
| 安全 | TPM 2.0 | 更强安全特性 |

## 🔧 在 OpenBMC 中使用 AST2700

### 设备树配置

```dts
// ast2700-evb.dts
#include "ast2700.dtsi"

/ {
    model = "ASPEED AST2700 EVB";
    compatible = "aspeed,ast2700-evb";

    chosen {
        bootargs = "console=ttyS0,115200 earlyprintk";
    };
};

& UART5 {
    status = "okay";
    console;
};

& KCS1 {
    status = "okay";
    aspeed,lpc-io-reg = <0xCA2>;
};
```

### Yocto 机器配置

```bash
# conf/machine/ast2700-evb.conf
require conf/machine/include/aspeed-bmc.inc

MACHINEOVERRIDES = ".:aspeed-bmc:ast2700:"

# AST2700 特定配置
UBOOT_MACHINE = "ast2700_evb_defconfig"
KERNEL_DEVICETREE = "aspeed-ast2700-evb.dtb"
```

### 构建命令

```bash
# 初始化 AST2700 环境
TEMPLATECONF=meta-aspeed/meta-ast2700/conf . openbmc/scripts/setup ast2700-evb

# 编译
bitbake obmc-phosphor-image
```

## 📡 常用外设接口

### I2C 总线

AST2700 提供 16 路 I2C 总线：

```bash
# 查看 I2C 总线
i2cdetect -l | grep ast2700

# 扫描总线上的设备
i2cdetect -y 0
```

### KCS 接口

KCS (Keyboard Controller Style) 用于 IPMI 通信：

```bash
# 内核日志查看 KCS 状态
dmesg | grep -i kcs
```

### UART 接口

8 路 UART，其中 UART5 通常用作 console：

```bash
# 连接串口
picocom -b 115200 /dev/ttyUSB0
```

## 🛡️ 安全特性

AST2700 提供增强的安全特性：

| 特性 | 说明 |
|------|------|
| Secure Boot | 安全启动链 |
| OTP | 一次性可编程区域 |
| eFuse | 密钥存储 |
| AES-256 | 硬件加密加速 |
| SHA-256 | 硬件哈希加速 |
| TPM 2.0 | 可信平台模块支持 |

### 启用 Secure Boot

```bash
# 在 U-Boot 中配置
setenv secureboot 1
saveenv
```

## 🐛 常见问题

### Q: AST2700 如何进入恢复模式？

1. 断电
2. 将 SW1 开关拨到 Recovery 位置
3. 上电
4. 通过 USB 或网络重新烧录固件

### Q: 如何查看 DDR 训练状态？

```bash
# U-Boot 中查看
bdinfo | grep -i ddr
```

### Q: 如何调试 I2C 总线？

```bash
# 使用 i2c-tools
i2cdetect -y 0
i2cget -y 0 0x50 0x00
i2cset -y 0 0x50 0x00 0xFF
```
